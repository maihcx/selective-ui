📌 CRV-UI Engine:

• Contract-Driven: All communication occurs through TypeScript interfaces — views do not depend on concrete implementations

• Retained DOM: DOM nodes are created once and updated incrementally, without Virtual DOM or diffing

• Virtualized Rendering: Fenwick Tree + windowing for 10k+ items with O(log n) height updates

• GLOBAL_SEUI singleton: The new version uses globalThis.GLOBAL_SEUI to prevent duplicate script tag loading ()

🆕 v1.3 Changes so with v1.2:

• Public API currently routed through globalThis.GLOBAL_SEUI singleton instead of module-level SECLASS

• Added registerPlugin / unregisterPlugin to the Public API

• Official FSM Lifecycle: NEW → INITIALIZED → MOUNTED → UPDATED → DESTROYED

• SelectBox now composes VirtualRecyclerView (virtual scroll) or RecyclerView

• Fenwick Tree replaces the previous simple scroll calculation

🔄 3. LIFECYCLE FSM (Finite State Machine)

stateDiagram-v2 [*] --> NEW: Constructor NEW --> INITIALIZED: init() note right of INITIALIZED bindedQueries = new Map() plugins = new Map() hooks containers created end note INITIALIZED --> MOUNTED: mount() note right of MOUNTED First successful bind() Active bound elements exist Ready for interactions end note MOUNTED --> UPDATED: update() UPDATED --> UPDATED: update() repeatable note right of UPDATED Triggered by: - bind() when already mounted - destroy() partial teardown - rebind() re-registration - Observer() new element detected end note MOUNTED --> DESTROYED: destroyAll() UPDATED --> DESTROYED: destroyAll() INITIALIZED --> DESTROYED: destroyAll() note right of DESTROYED bindedQueries cleared plugins cleared EAObserver disconnected All hooks cleared end note DESTROYED --> [*]

📌 FSM Guards (Idempotency):

• init(): no‑op if the state is not NEW

• mount(): only transitions INITIALIZED → MOUNTED; otherwise a no‑op

• update(): allowed in MOUNTED and UPDATED states; always emits onUpdate

• destroy(): idempotent if already DESTROYED; clears all hooks

• Hook exceptions are caught by handleHookError() — without breaking the lifecycle flow

📌 Advanced Virtual Scroll compared to v1.2:

• Fenwick Tree: simple formula replacement — supports dynamic/variable heights

• Adaptive estimator: uses running average of measured items instead of fixed estimateItemHeight

• Anchor correction: maintains scroll position stability when heights change after measurement

• Suspend/Resume: batch operations can pause virtualization to avoid feedback loops

• Re-entrancy guard: the updating flag prevents recursive calls to updateWindowInternal

🎯 Fenwick Tree Formulas:

  • offsetTopOf(i) = fenwick.sum(i) → prefix sum of heights[0..i-1]
  • findFirstVisible(scrollTop) = fenwick.lowerBoundPrefix(scrollTop)
  • overscanPx = opts.overscan × getEstimate()
  • adaptiveEstimate = measuredSum / measuredCount (khi measuredCount > 0)
  • totalHeight(count) = fenwick.sum(count)
  • windowHeight(s,e) = fenwick.rangeSum(s+1, e+1)

📌 Detailed Memory Management:

• EAObserver disconnect: Pause in destroyElement to avoid re-bind when the DOM changes

• deInit() vs destroy(): deInit is SelectBox-specific cleanup, destroy is Lifecycle teardown

• Plugin teardown: calls both plugin.destroy() and plugin.onDestroy()

• VirtualRecyclerView: cancel rAF, remove scroll listener, disconnect ResizeObserver, remove scaffold nodes

• BinderMap: removed from WeakMap after destroy — GC can reclaim references
